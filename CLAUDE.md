# krci-docs

Docusaurus v3 static site for [docs.kuberocketci.io](https://docs.kuberocketci.io). Package manager: **yarn** (v1.22). Node ≥ 18 required.

## Commands

```bash
yarn install          # install dependencies
yarn start            # dev server with live reload (localhost:3000)
yarn build            # production build → build/  (postbuild runs fix-llms-blog-urls.js)
yarn serve            # serve the built output locally
yarn typecheck        # tsc (TypeScript check of src/)
yarn lint             # ESLint on src/
yarn lint:fix         # ESLint auto-fix
yarn format           # Prettier on src/
yarn format:docs      # Prettier on docs/
yarn spell-check      # cspell on docs/** and blog/** (config: cspell.config.yaml)
yarn clear            # clear Docusaurus cache
```

CI runs `npm ci` (not yarn) then: `npm run build` → `npm run spell-check` → `npm run typecheck`.

`onBrokenLinks: 'throw'` and `onBrokenMarkdownLinks: 'throw'` — broken links fail the build.

## Architecture

```
docs/                    — current (unreleased) content; this is what you edit
versioned_docs/          — frozen snapshots (3.12, 3.13, 3.14); do NOT edit
versioned_sidebars/      — sidebar configs for each frozen version
versions.json            — list of frozen versions ["3.14","3.13","3.12"]
sidebars.ts              — sidebar config for current docs/
sidebarsFAQ.ts           — sidebar for faq/
faq/                     — FAQ section (separate plugin instance)
blog/                    — blog posts (Markdown/MDX)
api/                     — Azure Functions backend for the static site
src/
  components/            — shared React components
  css/                   — custom.css (theme overrides)
  features/              — landing-page feature blocks
  pages/                 — custom Docusaurus pages
  plugins/               — custom Docusaurus plugins
    recent-blog-posts/   — exposes 3 most recent posts as global data for landing page
  theme/                 — Docusaurus theme swizzle overrides (Blog*, DocItem, Footer, Root.tsx)
static/                  — static assets (images, icons, llms.txt backup)
scripts/
  fix-llms-blog-urls.js  — postbuild: rewrites dated blog URLs in llms.txt to slug-based canonical paths
docusaurus.config.ts     — main config: URL, presets, plugins, navbar, footer, Algolia, versioning
```

### Docs versioning

- **Edit `docs/`** for new or current-cycle content. The current dev version is labelled `3.15-dev` (path prefix: `next`).
- **`versioned_docs/version-X.Y/`** are frozen snapshots generated by `yarn docusaurus docs:version X.Y`. Do not hand-edit them; they are locked to the release they represent (currently `3.14`, `3.13`, `3.12`).
- To cut a new version: `yarn docusaurus docs:version <semver>` — this copies `docs/` to `versioned_docs/version-<semver>/`, updates `versions.json`, and creates `versioned_sidebars/version-<semver>-sidebars.json`.
- The first entry in `versions.json` is the default `/docs` version (latest). Unreleased content is served at `/docs/next/`.

### Content sections

`docs/` is subdivided into: `quick-start/`, `operator-guide/`, `user-guide/`, `developer-guide/`, `use-cases/`, `api/`, `assets/`. Top-level files include `about-platform.md`, `basic-concepts.md`, `glossary.md`, `supported-versions.md`.

### Custom plugins & LLMs support

- `docusaurus-plugin-llms` generates `llms.txt` and `llms-full.txt` at build time (excludes `faq/**` and `api/**`).
- `scripts/fix-llms-blog-urls.js` (postbuild) rewrites any dated blog paths to slug-based canonical paths in those files.
- `src/plugins/recent-blog-posts/index.js` — custom plugin that injects the 3 most recent blog posts as global Docusaurus data.

### Spell-check

`cspell.config.yaml` holds the project word allowlist. Add new technical terms there rather than suppressing the CI check.
